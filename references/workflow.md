# JD-Resume Optimizer 详细交互流程

## 第一步：岗位解析 (JD Analysis)

### 输入接收
- 用户可发送：纯文本 JD、JD 截图（图片）、JD 链接（网页）
- 如果是图片/链接，使用 image/web_fetch 工具提取文字内容

### 关键词提取维度
1. **硬技能关键词**（必须出现在简历中的工具/技术）
   - AI 工具：Claude Code, Cursor, Codex, Kiro, LangGraph, etc.
   - 技术栈：React, FastAPI, PostgreSQL, Redis, Python, etc.
   - 方法论：AI First, Align-Build-Demo, Agent Infra, etc.

2. **软技能关键词**
   - 业务 Sense, 快速迭代, 跨部门协作, 0-1 经验

3. **经验要求**
   - 年限：5-10 年
   - 行业：互联网、金融、AI
   - 背景：工程背景、产品背景

4. **加分项**
   - 独立项目、GitHub Stars、个人作品

### 输出格式
```
📋 岗位分析结果
━━━━━━━━━━━━━━━━━━━━
🎯 岗位本质：XXX
🔑 核心关键词：[列表]
💪 必备能力：[列表]
➕ 加分项：[列表]
⚠️ 不匹配预警：[列表]
```

## 第二步：简历输入 (Resume Input)

### 接收方式
- PDF 文件（使用 pdf 工具解析）
- Word 文件（提示用户粘贴内容）
- 纯文本（直接接收）
- 截图（使用 image 工具 OCR）

### 解析要点
提取以下结构化信息：
- 头部：姓名、联系方式、求职方向
- 核心能力/技能
- 项目经历（名称、技术栈、成果）
- 工作经历（公司、时间、职责、成果）
- 教育经历

## 第三步：匹配与调整 (Matching & Adjustment)

### 匹配度分析
对比 JD 关键词与简历内容，标记：
- ✅ 已覆盖：简历中已有且角度正确的内容
- ⚠️ 角度偏差：经历存在但叙述角度不匹配 JD
- ❌ 缺失：简历中完全没有相关内容

### 微调原则（不重写）
1. **保留事实**：原有经历的时间、公司、职责不变
2. **调整叙述角度**：
   - 例："数据分析师" → "数据产品化 / 内部工具构建"
   - 例："写 SQL 报表" → "搭建数据可视化看板产品"
3. **补充关键词**：在技能描述中自然融入 JD 关键词
4. **不硬编**：缺失的经历如实标注，建议用户补充

## 第四步：用户确认 (User Confirmation)

### 展示内容
```
🔍 优化方案预览
━━━━━━━━━━━━━━━━━━━━
📌 JD 关键词：[提取的关键词列表]

📊 匹配度分析：
   ✅ 已覆盖：X 项
   ⚠️ 角度偏差：X 项（具体说明）
   ❌ 缺失：X 项（建议补充）

📝 计划修改：
   1. [具体修改点 1]
   2. [具体修改点 2]
   ...

❓ 请确认是否继续生成？（是/否/调整）
```

### 交互逻辑
- 用户确认 → 进入第五步
- 用户拒绝 → 询问修改意见，返回第三步
- 用户要求调整 → 记录调整需求，返回第三步

## 第五步：最终输出 (Final Output)

### 输出内容
1. **优化后的简历**（结构化文本，供用户审阅）
2. **HTML 文件**：浏览器打开，可编辑，可打印
3. **PDF 文件**：直接投递用
4. **长截图**：手机查看/微信转发

### 文件位置
默认输出到 `/mnt/agents/output/`
命名格式：`简历_{姓名}_{岗位关键词}.{格式}`

### 使用脚本
- HTML 生成：`python3 scripts/generate_html_resume.py --input data.json --output resume.html`
- PDF 转换：`python3 scripts/html_to_pdf.py --input resume.html --output resume.pdf`
- 长截图：`python3 scripts/screenshot_resume.py --input resume.html --output resume.png`

### 输出页面要求（必须遵守）
- **去掉生成时间、文件信息**：页面顶部和底部不显示任何元信息
- **去掉底部链接**：不显示"下载"、"链接"等提示
- **截图去空白**：使用 PIL 自动裁剪底部空白，精确到内容末尾
- **排版固定**：使用 `assets/resume-template.html` 作为固定模板，后续仅允许调整颜色变量，不改布局
- **生成步骤**：先生成 HTML → 转 PDF → 转截图，不直接给 PDF 截图

## 重要提醒

### 不欺骗原则
- 不编造不存在的经历
- 不夸大未做过的事情
- 技能关键词只在确实掌握的前提下添加
- 对不确定的内容标注为"建议补充"

### 格式偏好
- 用户偏好：避免表格形式，文字直接输出
- 风格：接地气、增强可信度、反对过于隆重
- 工作描述范式："独立负责...覆盖...深入业务前线..."
